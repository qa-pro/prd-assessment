# PRD Test/QA Checklist

> 面向测试/QA 视角评审 PRD，确保需求描述支持测试计划、用例设计与质量度量，而无需在 PRD 写具体测试脚本。

## 场景覆盖（P0）
- [ ] PRD 列出所有业务流程、异常分支、跨系统场景，QA 可据此枚举测试路径。
- [ ] Scope in/out、版本拆分、依赖前提清晰，便于确定回归边界。
- [ ] 角色、权限、地域、终端等差异场景被完整描述。

## 需求判定口径（P0）
- [ ] 每个需求具备可量化或可验证的验收标准（Success Criteria）。
- [ ] PRD 指明成功/失败的业务指标或可观测信号（日志、埋点、接口返回码）。

## 数据与接口（P0）
- [ ] 输入数据范围、合法/非法值、边界条件在 PRD 中描述。

## 状态与异常（P0）
- [ ] 所有状态转移、重试、补偿流程在 PRD 描述，QA 能判定预期行为。
- [ ] 错误提示、兜底策略、告警机制写明，方便验证体验与告警链路。
- [ ] 灰度/回滚触发条件、限制条件记录，以备编写相应用例。

## 量化指标与监控（P1）
- [ ] 关键指标（业务/技术）在 PRD 中定义公式与采集方式，便于上线验证。
- [ ] 埋点/监控需求列出事件、字段、触发条件及归属，QA 可同步校验。

## 性能与容量（P1）
- [ ] 峰值负载、并发、批量处理窗口、消息体量等指标被描述。
- [ ] 性能/稳定性风险场景（大数据、弱网、延迟依赖）注明，QA 可提前设计专项测试。
- [ ] 需开展压测、基准测试或混沌测试的范围、目标、数据准备方式在 PRD 中说明。

## 多终端与兼容性（P1）
- [ ] 支持的浏览器、OS、App 版本、分辨率、设备类型列出。
- [ ] 特殊容器、嵌入式场景、离线/弱网等限制说明齐全。
- [ ] 国际化、时区、语言、货币等差异化要求写明，QA 可覆盖。

## 安全、权限与合规（P2）
- [ ] 需要验证的认证、授权、审计、审批流程写清楚。
- [ ] 敏感信息、隐私字段、脱敏策略、数据留存周期标注，便于测试确认。
